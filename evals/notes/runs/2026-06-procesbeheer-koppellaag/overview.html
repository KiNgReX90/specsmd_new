Tweede run, een grotere en bredere intent, dezelfde TDD-bouwer. De zware AI-DLC-controle wordt nu Pareto-gedomineerd: aidlc-turbo levert betere code met 73% minder documentatie. De lean-controle is opnieuw de slechtste build én zakt voor de caveat-gate — precies waar ze caveats liet vallen.

aidlc-turbo domineert de zware controle — op beide assen

8,06 vs 7,67 op kwaliteit, en 73% minder regels (1.282 vs 4.789). Het documentvolume kocht geen correctheid of getrouwheid; de gedisciplineerde beslissingen-ledger + getrimde stories bouwden betere code met veel minder om te reviewen. De schoonste bevestiging van de hypothese die je kunt krijgen.

Eén dispuut, heradjudiceerd — niet stil gemiddeld

De correctheids-lenzen scoorden inferno-pb 3,2 en 4,8 (load-bearing kolomfout), de idioom-lens 7,8 ("intern consistent"). De caveats noemen BPS_PROCES_TYPE expliciet als discriminator en markeren BPS_INDELING_SOORT als retirend → twee correctheidslezingen + caveat-bewijs dragen het. De lage scores blijven staan; de spreiding zélf is het signaal van een dubbelzinnige doc.

03 · Waar de documentvorm telt — en waar niet

Twee scherpe bevindingen

Niet elke bolt is even gevoelig voor de documentvorm, en de beste slanke vorm bleek intent-afhankelijk. Beide nuances sturen de aanbeveling.

De service-bolt discrimineert · de UI-bolt niet

spreiding pb 6,59→8,01  ·  kl 7,32→8,12

Documentvorm telt het meest waar er échte businesslogica is om goed te krijgen. Op de logica-zware procesbeheer-service waaieren de armen ver uit; op de delegatie-zware koppellaag-UI (dunne laag over een bestaande business-API) klitten ze samen. Het caveat-verlies van de lean-doc bijt het hardst precies daar waar de build het moeilijkst is — en is vrijwel onzichtbaar waar de UI alleen delegeert.

turbo-a herhaalt #1 niet → aidlc-turbo is de robuuste default

BTW: turbo-a 8,65 (1e)  →  hier 7,60 (3e)

turbo-a wás de beste build op run #1, maar zakt hier naar de derde plek, getrokken door zijn kl-build: een handgerolde HTML-encoder die dubbel-encodet tegen DomUI's eigen escaping. De gefuseerde één-item-vorm is gevoeliger voor per-bolt bouwer-slips. aidlc-turbo is de enige arm die op beide stacks sterk is (8,01 / 8,12) — daarom de veilige cross-intent keuze.

04 · Het oordeel

Documenteconomie vs. bouwkwaliteit

Documenteconomie uit Stage A (0–10, build-loos) tegen bouwkwaliteit uit het jurypanel. Elke gate-geschikte slanke arm blijft binnen ε = 0,5 van de controle (7,17) terwijl ze 73–83% van de docs snoeit.

Aanbeveling

Neem aidlc-turbo als standaard slanke vorm — beste code, sterk op beide stacks, kleinste migratie vanaf het huidige AI-DLC (ceremonie eruit, getrimde stories + beslissingen-ledger erin).

Vouw vóór adoptie twee gates in de flow: een idioom/lint-gate (ASCII-only, strings in .properties, geen dubbele encoding, geen cross-layer enum-plaatsing) en een caveat-inline-check — zodat geen arm herhaalt wat dit panel signaleerde. Daarna de derde poot draaien: de financial-middleware cross-app check (4 intents).
